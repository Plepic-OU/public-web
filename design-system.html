Design Decisions

  • Single hue (137°) for all greens — no hue drift between shades.
  • System saturation: 73%. Surface, light, brand, and dark all share S=73%.
  • Vivid exception: S=100%. Decorative punch only — dots, fills, dark-mode headings. Never as text on light.
  • Accent at H=15°, S=73% — split-complementary of 137°, saturation matched.
  • One accent per viewport. A CTA button and an urgency badge must never be visible simultaneously.
  • No accent variants. Use opacity for hover/active states.
  • Green = structural identity. Logo, headings, navigation, badges, links.
  • Orange = action warmth. CTA buttons and urgency indicators only.

8 — Rules & Anti-Patterns

Do

  • Use --green-brand for text on light backgrounds (AA)
  • Use --green-dark for body text needing AAA
  • Use --green-vivid for decorative elements and dark-mode headings
  • Use --accent for exactly one element per viewport — a CTA button OR an urgency badge, never both
  • Use asymmetric grids (1.4fr / 0.6fr), not 50/50
  • Use spacing rhythm — tight groupings, generous separations
  • Set line-height: normal on compact components
  • Light mode only. Dark sections for emphasis, not a toggle.

Don't

  • No --green-vivid as text on light — fails AA (2.5:1 on cream)
  • No multiple accent-colored elements in one viewport — one CTA button or one urgency badge or one accent dot, never two
  • No accent hover variants — use opacity for states
  • No cyan, neon, glassmorphism, gradient text, glow effects
  • No all-caps headings (except mono labels and trust bar)
  • No centered-everything layouts on desktop
  • No SVG blur/glow filters
  • No AI slop aesthetics (2024-2025 generic look)
  • No #8a8a80 as readable text — use --text-3 (#6b6b60) minimum
