The Definition

What is a coding agent?

An LLM that uses tools in a loop with persistent context. It thinks, acts, and observes — until the task is done. Claude Code is the canonical example: it lives in your terminal, reads your codebase, runs tests, and ships changes.

Why Claude Code

The tool developers actually love

Claude Code tops developer satisfaction surveys for a reason. Four reasons, actually.

Early Mover Advantage

Anthropic shipped Claude Code in February 2025 — 11 months before Cursor's terminal agent. The harness, hooks, and MCP ecosystem are years ahead.

Terminal-Native

No IDE sandbox. Full bash, full filesystem, full git. Compose with any CLI tool you already use — gh, docker, kubectl, your own scripts.

A Mature Harness

Hooks, sub-agents, MCP servers, custom slash commands, skills. The agent is genuinely extensible — not a chatbot with a fancy UI.

The Best Models

Claude Opus and Sonnet lead every coding benchmark that matters. Claude Code is built by the team training the models — tight feedback loop, fast iteration.

The Shift

Why we bet on Claude Code

Every loop iteration is informed by the previous “observe” — that is reflection. The iteration itself creates learning. Claude Code was the first tool to make this loop feel native.

“After writing my first Claude Code loop, I couldn't sleep. Every repetitive task, every debugging session, every code review — I could see how each one could be transformed.

Once you build your first loop, you stop thinking about AI as a chatbot. You start seeing it as a collaborator that improves with each iteration. That's the shift we teach.”
